1
Contexte & Objectif

Le site fleursdenila.fr est aujourd'hui un site vitrine entièrement statique (HTML, Tailwind CSS, JavaScript vanilla), hébergé sur un serveur Apache. Il bénéficie d'un travail SEO soigné : balises Schema.org, balises canoniques, sitemap optimisé, règles de cache avancées dans le .htaccess, et des temps de chargement très performants.

L'objectif de ce projet est d'y intégrer une boutique en ligne sans altérer ni la structure existante, ni le référencement naturel acquis.

Exigence exprimée par Manon : la boutique doit s'afficher sur le même site, à la même URL fleursdenila.fr, avec le même design — sans redirection vers un domaine tiers ni rupture d'expérience pour le client.

3
Solutions étudiées & Solution retenue

Six solutions ont été analysées. Les plateformes SaaS (Shopify, Ecwid, Shoprocket) ont été écartées car aucune ne répond simultanément aux deux critères non négociables : même site/URL et gestion de créneaux avec capacité limitée (1 livraison par créneau). La solution retenue est un développement full custom.

4
La solution retenue : développement full custom

La boutique est développée en JavaScript vanilla côté frontend, avec une API PHP hébergée directement sur le serveur LWS existant de fleursdenila.fr. La base de données MySQL (incluse dans l'hébergement LWS) gère les produits, stocks et créneaux. Stripe gère le paiement. Tout est hébergé au même endroit — aucun service externe supplémentaire. Le client ne quitte jamais fleursdenila.fr.

Même site, même URL, même design Aucun sous-domaine, aucune redirection. SEO et charte graphique intacts.
Créneaux avec capacité limitée 1 livraison par créneau. Un créneau pris disparaît en temps réel pour les autres clients.
Variantes & personnalisation Couleurs, tailles S/M/L, message personnalisé à joindre au bouquet.
Gestion des stocks Décrémentation automatique, désactivation produit à stock zéro.
Retrait boutique + livraison Les deux modes disponibles avec règle de livraison offerte au-dessus d'un seuil.
Back-office dédié Vue journée du lendemain, gestion produits/stocks, export CSV comptable.
Zéro abonnement mensuel Aucune plateforme SaaS. Uniquement les frais Stripe sur les ventes réalisées.
Sécurité complète Signature webhook Stripe, authentification back-office, protection RGPD (données EU).

5
Plan d'intégration — Solution full custom
1
Mise en place de l'infrastructure (MySQL LWS + Stripe)
Création de la base de données MySQL sur l'hébergement LWS existant, tables produits / stocks / créneaux / commandes, configuration compte Stripe.
2
Développement API PHP — produits & stocks
Scripts PHP (PDO + requêtes préparées) : liste produits, variantes, disponibilité stock. Décrémentation automatique à chaque commande confirmée.
3
Développement API PHP — gestion des créneaux
Scripts PHP : créneaux disponibles du lendemain, réservation temporaire (10 min anti-doublon via transaction MySQL), confirmation définitive. 1 livraison maximum par créneau.
4
Intégration Stripe Checkout + webhook PHP
Session de paiement avec prix recalculé côté serveur PHP (non falsifiable). Webhook.php vérifié : paiement confirmé → stock décrémenté, créneau définitivement bloqué, email envoyé.
5
Développement frontend — boutique & panier
Page boutique.html : catalogue produits, variantes (couleur, taille, message), panier JS, règle livraison offerte au-dessus du seuil défini.
6
Développement frontend — sélecteur de créneaux
Calendrier JS : affichage des créneaux disponibles en temps réel, grisage automatique des créneaux pris, choix livraison à domicile ou retrait boutique.
7
Développement back-office Manon
Page admin protégée par session PHP (login/mot de passe sécurisé) : vue tournée du lendemain, gestion produits/stocks, historique commandes, export CSV comptable.
8
Sécurité & conformité RGPD
Vérification signature Stripe, requêtes PDO préparées (anti-injection SQL), rate limiting .htaccess, sanitisation des données, données hébergées en France (LWS), mise à jour politique de confidentialité.
9
Intégration visuelle charte Fleurs de Nila
Adaptation complète aux couleurs, polices et espacements du site existant — cohérence parfaite entre vitrine et boutique.
10
Tests end-to-end & mise en ligne
Tests complets : paiement, double réservation simultanée, rupture de stock, webhook, export, back-office. Mise en production sur fleursdenila.fr.
11
Formation Manon — back-office (1 h)
Prise en main complète : consulter la tournée du lendemain, gérer les stocks, modifier un produit, exporter pour le comptable.
