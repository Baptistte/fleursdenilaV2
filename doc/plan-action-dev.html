Architecture technique

fleursdenila.fr (hébergement LWS) │ ├── boutique.html → catalogue produits + panier JS ├── checkout.html → récap commande + sélecteur créneaux + Stripe ├── confirmation.html → page post-paiement ├── admin/ │ ├── index.php → login (session PHP) │ ├── produits.php → CRUD produits + stocks │ └── tournee.php → vue livraisons du lendemain │ └── api/ ├── products.php → GET liste produits + stocks ├── slots.php → GET créneaux dispo / POST réservation / DELETE expiration ├── checkout.php → POST création session Stripe (prix serveur) └── webhook.php → POST Stripe → confirme créneau + décrémente stock MySQL LWS ├── products (id, nom, description, prix, stock, actif, variantes JSON) ├── slots (id, date, heure, statut, order_id, expires_at) └── orders (id, stripe_id, produits JSON, slot_id, client JSON, created_at) Back-office commandes / paiements / exports → Stripe Dashboard (natif, zéro dev)

Résumé : Stripe Dashboard couvre nativement tout ce qui touche à l'argent (commandes, paiements, remboursements, exports, virements) — sans une ligne de code. Le développement custom couvre tout ce qui est spécifique au métier de Manon : créneaux, stocks, catalogue et tournée du lendemain.

Phase 1 — Infrastructure & base de données

TâcheDétailTypeTemps
Création base MySQL LWS Création BDD via cPanel LWS, utilisateur dédié avec droits limités (SELECT/INSERT/UPDATE/DELETE uniquement — pas de DROP) MySQL 1h
Création des tables Tables products, slots, orders avec index, contraintes et champ expires_at pour les réservations temporaires MySQL 2h
Jeu de données initial Insertion des 5-15 premiers produits avec variantes (couleurs, tailles S/M/L), stocks initiaux et génération des créneaux de la semaine suivante MySQL 2h
Configuration compte Stripe Création ou liaison compte Stripe de Manon, récupération clés API (test + prod), configuration webhook endpoint sur le serveur LWS Stripe 1h
Total phase 1 6h

Phase 2 — API PHP

TâcheDétailTypeTemps
Classe de connexion BDD Singleton PDO avec requêtes préparées systématiques, gestion des erreurs, fichier de config hors racine web (config.php protégé par .htaccess) PHP 2h
api/products.php GET : liste produits actifs avec variantes et stock. Filtre automatique des produits à stock 0. Réponse JSON. PHP 3h
api/slots.php — lecture GET : liste des créneaux disponibles du lendemain (statut = libre). Nettoyage automatique des réservations expirées (expires_at dépassé). PHP 3h
api/slots.php — réservation temporaire POST : réservation d'un créneau pour 10 min via transaction MySQL (verrouillage SELECT FOR UPDATE pour éviter les doublons simultanés). Retourne un token de session. PHP 4h
api/checkout.php POST : vérification stock + créneau toujours disponible, calcul prix côté serveur (jamais depuis le client), création session Stripe Checkout avec métadonnées commande. Stripe 4h
api/webhook.php POST Stripe checkout.session.completed : vérification signature HMAC, confirmation définitive du créneau en base, décrémentation stock, envoi email Manon (mail() ou SMTP LWS). Stripe 4h
Total phase 2 20h

Phase 3 — Frontend JS (côté client)

TâcheDétailTypeTemps
Page boutique — catalogue Chargement dynamique des produits via api/products.php, affichage des variantes (couleur, taille, message), badge "rupture de stock" automatique. JS 6h
Panier JS Ajout/suppression/modification quantité, persistance localStorage, calcul sous-total, règle livraison offerte au-dessus du seuil défini, badge compteur panier. JS 7h

Page confirmation Page post-paiement affichant le récapitulatif de commande et le créneau confirmé. Lecture des paramètres Stripe retournés dans l'URL. JS 2h
Total phase 3 27h

Phase 4 — Back-office Manon

La gestion des commandes, paiements et exports comptables est assurée nativement par Stripe Dashboard — aucun développement nécessaire pour cette partie. Le back-office custom couvre uniquement la gestion produits/stocks et la vue tournée du lendemain.
TâcheDétailTypeTemps
Authentification admin Page login PHP avec session sécurisée (password_hash / password_verify), protection de toutes les pages admin, déconnexion automatique après inactivité. PHP 2h
Gestion produits & stocks Liste des produits, ajout / modification / désactivation, mise à jour du stock en un clic. Interface simple, utilisable sans formation technique. PHP 4h
Vue tournée du lendemain Page affichant la liste des livraisons du lendemain triée par créneau horaire : client, adresse, produits commandés, mode (livraison/retrait). Imprimable. PHP 2h
Total phase 4 8h

Phase 5 — Sécurité & conformité

TâcheDétailTypeTemps
Protection fichiers sensibles Règles .htaccess pour bloquer l'accès direct à config.php, au dossier api/ hors appels AJAX, et à l'admin hors session active. Sécu 1h
Vérification webhook Stripe Validation obligatoire de l'en-tête Stripe-Signature (HMAC SHA-256) dans webhook.php — toute requête non signée est rejetée. Sécu 1h
Anti-injection SQL 100 % des requêtes SQL via PDO avec paramètres préparés — aucune concaténation de variable dans les requêtes. Sécu 1h
Rate limiting réservations Limitation à 1 réservation de créneau active par session — blocage des tentatives d'accaparement automatisé des créneaux. Sécu 2h
Sanitisation des données Échappement systématique des données affichées en back-office (htmlspecialchars), validation des types reçus dans les API (dates, entiers, chaînes). Sécu 1h
RGPD Données clients hébergées en France (LWS). Mise à jour de la politique de confidentialité existante (mention du traitement des données de commande et durée de conservation). Sécu 1h
Total phase 5 7h
