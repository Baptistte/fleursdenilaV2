Récapitulatif — Pourquoi le développement full custom

Fleurs de Nila — fleursdenila.fr — Avril 2026

1. Les critères non négociables de Manon

Critère Détail Priorité
Même site, même URL La boutique doit s'afficher sur fleursdenila.fr, sans redirection ni sous-domaine Indispensable
Créneaux de livraison avec limite 1 seule livraison par créneau — un créneau pris doit être bloqué en temps réel pour les autres clients Indispensable
Variantes produits Couleur, taille (S/M/L), message personnalisé à joindre au bouquet Indispensable
Gestion des stocks Ruptures fréquentes — désactivation automatique produit à stock zéro Indispensable
Retrait boutique + livraison Les deux modes disponibles, livraison offerte au-dessus d'un seuil Indispensable
Back-office simple Vue tournée du lendemain, gestion produits/stocks, export CSV comptable Indispensable
Zéro abonnement mensuel fixe Pas de plateforme SaaS à payer chaque mois Souhaité
Design cohérent avec le site existant Même charte graphique, même police, mêmes couleurs Souhaité

2. Comparatif des solutions étudiées

Toutes les solutions du marché ont été étudiées et testées au regard des critères ci-dessus. Deux critères ont été bloquants pour chaque solution SaaS : la gestion des créneaux avec capacité limitée et/ou l'intégration sur le même site.
Solution Même site Créneaux limités Stock Variantes Coût mensuel Verdict
Stripe Payment Links ✗ ✗ ✗ ✗ 0 € + comm. Écarté
Shopify Buy Button Partiel ✗ SDK déprécié jan. 2025 ✓ ✓ ~29 €/mois Écarté
Ecwid ✓ ✗ non supporté ✓ ✓ ~19 €/mois Écarté
Snipcart ✓ ✗ note simple seulement Basique ✓ 10 $/mois + 2 % Écarté
Shoprocket ✓ ✗ note simple seulement ✓ ✓ ~19 $/mois Écarté
WooCommerce + plugin ✗ migration complète ✓ natif ✓ ✓ ~15 €/mois Risque SEO
Full custom (JS + PHP + MySQL) ✓ ✓ natif ✓ ✓ 0 € ★ Retenu

3. Pourquoi le full custom répond à tout

Critère Manon Comment le full custom y répond
Même site / même URL La boutique est développée directement dans le site existant — aucun service tiers, aucune redirection
Créneaux limités (1 livraison / créneau) API PHP + MySQL : chaque créneau est verrouillé en base à la réservation. Deux clients simultanés ne peuvent pas prendre le même créneau (transaction MySQL)
Variantes produits Gérées en base de données, affichées dynamiquement en JS, transmises à Stripe en métadonnées
Gestion des stocks Stock décrémenté côté serveur PHP à chaque webhook Stripe confirmé — jamais côté client (non falsifiable)
Retrait boutique + livraison Deux modes configurables dans la base de données, seuil de livraison offerte calculé côté serveur
Back-office simple Page admin.html dédiée, protégée par session PHP — vue tournée du lendemain, CRUD produits, export CSV
Zéro abonnement mensuel MySQL et PHP inclus dans l'hébergement LWS existant — aucun service SaaS supplémentaire
Design cohérent HTML/CSS développé directement avec la charte Fleurs de Nila (couleurs, polices, espacements identiques)
Sécurité paiement Stripe Checkout : les numéros de carte ne transitent jamais par le serveur. Webhook signé et vérifié côté PHP
Données hébergées en France LWS est un hébergeur français — toutes les données clients restent sur des serveurs français (conformité RGPD facilitée)
